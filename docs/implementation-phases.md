# 实施阶段与完成门槛

1. **项目骨架**：前后端可启动，健康检查、格式化、静态检查和测试通过。
2. **数据库与基础业务**：ER、迁移、任务/日历/热词/日志 API 和数据库测试通过。
3. **语音链路**：浏览器 PCM、WebSocket、ASR 适配器、临时/最终结果和错误状态通过测试。
4. **意图与可靠执行**：Schema、追问、风险、确认、事务、验证和撤销通过故障测试。
5. **校园知识库**：四类文档解析、检索、真实出处、证据不足和通知转草稿通过测试。
6. **术语纠错**：热词候选、关键字段保护、分级确认和记录通过测试。
7. **评测与优化**：指标由原始数据脚本生成，十个端到端场景通过。
8. **部署与文档**：Compose、CI、示例数据、启动说明和演示脚本完成。

阶段完成证据必须包括变更文件、测试命令、测试结果和明确的遗留问题。
