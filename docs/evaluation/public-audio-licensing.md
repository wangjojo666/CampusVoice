Common Voice 的“有条件允许”只覆盖本地评测和汇总指标证据链。若将来需要在公开演示中播放某段真人原声，应单独进行权利评审，不能从本表推导出播放许可。

## 为什么 WenetSpeech 不能按 Apache-2.0 使用

WenetSpeech 官方仓库页面显示 `Apache-2.0`，但同一仓库要求下载者前往官方数据站阅读数据许可。官方数据站对**数据**增加了非商业用途限制，并明确说明音频版权仍属于原始来源。因此：

1. `Apache-2.0` 只能作为仓库代码许可记录，不能填入 `speech_license` 代表音频授权。
2. `CC BY 4.0` 与同页的 non-commercial 限制并不形成一个可以自行扩张解释的宣传许可。
3. 项目宣传是否属于商业用途可能随主办方、发布渠道和获益方式变化；在没有书面澄清前一律按不允许处理。

## 环境声候选与逐条筛选

[Freesound 官方 FAQ](https://freesound.org/help/faq/) 可作为环境声的候选目录，但 Freesound 不是一个对所有条目采用统一许可的数据集。本阶段**不下载任何环境声**；实际准备数据时必须逐条核验声音详情页。

只接受满足以下全部条件的条目：

- 详情页明确显示 `CC0-1.0`，或明确显示 `CC BY 4.0` 且项目能够完整履行署名义务。
- 记录条目标题、作者/上传者、原始页面 URL、精确许可名称与许可 URL、许可核验日期、下载日期和文件 SHA-256。
- `CC BY 4.0` 条目在数据卡、报告和演示致谢中保留作者、标题、来源、许可和修改说明。
- 内容与目标噪声类别相符：食堂/多人交谈背景、走廊/脚步、户外风声/道路声；条目中若存在可辨识独白、姓名或其他个人信息则拒绝。
- 页面、下载文件与 manifest 中的许可信息一致；存在冲突时停止使用并标为 `NOT_RUN`。

以下条目一律拒绝：

- `CC BY-NC`、`NC`、`ND`、`Sampling+`、仅限学术用途、许可版本不明或来源不明。
- 来自随机短视频、直播、网课、播客、网盘转载或无法追溯到权利人的音频。
- 仅凭搜索结果、文件名或上传者描述推断许可的音频。
- AI/TTS 生成但被描述为真人录音的音频。

## 准入检查清单

每个来源在混音或推理前都必须满足：

1. 官方或第一方页面可访问，并保存数据集/条目的精确版本和 URL。
2. 使用场景（本地评测、公开演示、宣传发布）落在许可范围内；模糊、冲突或需要解释时默认拒绝。
3. `speech_source`、`speech_dataset_version`、`speech_license`、`noise_source`、`noise_license`、`noise_license_verified_at` 等必填 provenance 字段完整，且与官方页面完全一致。
4. 音频只保存在 Git 忽略的本地生成目录；CI 不下载数据集、音频或大型模型。
5. Common Voice 不做说话人身份识别，不重新托管或分享数据；报告只引用汇总结果和数据卡链接。
6. 所有 `CC BY 4.0` 环境声具备完整署名记录。
7. 同一匿名说话人不同时进入调参与测试集合。
8. 缺失数据、授权字段、文件或实际推理记录时返回 `NOT_RUN`，不得生成指标占位值。

## 报告与宣传措辞

在真实推理完成、报告可复现且许可闸门通过后，可以写：

> 演示评测使用公开授权真人语音，并叠加可复现的模拟校园噪声；这不等同于真实校园实地测试。

若使用 Common Voice，应同时写明 `zh-CN`、26.0、具体 split、样本数、匿名说话人数、噪声类别、SNR、模型版本、设备、测试日期和指标计算方式。实际结果产生前不得填入 CER、完整句准确率、延迟或“提升”数字。

以下表述在没有对应真实证据前禁止使用：

- “经真实校园环境验证”
- “真实学生授权录音测试”
- “所有方言表现领先”
- “复杂校园环境高准确率”
- “行业领先”
- “自动监测校园通知”

WenetSpeech 在当前状态下也不得出现在宣传结果的测试集列表中。取得书面许可并重新通过本闸门后，才可更新该结论。
